## 社区发帖 FK 约束 + 全局异常处理器修复（23:50）

### Bug: POST /api/prompt/community → 500 FK constraint violation

**根因：** `user_prompts.category_id` 表定义为 `BIGINT DEFAULT 0`，FK 引用 `prompt_categories(id)`。但 `prompt_categories` 没有 id=0 的行（AUTO_INCREMENT 从 1 开始）。MyBatis Plus 对 null 字段默认不写入 INSERT，MySQL 使用 DEFAULT 0 → FK 失败。

**修复：**
1. **Flyway V12 迁移**：`ALTER TABLE user_prompts MODIFY category_id BIGINT NULL DEFAULT NULL`
   - InnoDB FK 约束对 NULL 值不强制执行，无需改动外键
2. **编译 + 重启验证**：社区发帖不带 category_id → 200 ✅

### Bug: 后端暴露完整 SQL 异常到前端

**根因：** `GlobalExceptionHandler.handleRuntimeException()` 直接 `e.getMessage()` 透传，SQL 异常继承自 RuntimeException。

**修复：**

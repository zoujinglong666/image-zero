## 分析链路 fallback 死循环修复（22:34）

**问题：** 智谱分析失败 → `analyzeWithOpenAI(imageUrl)` → 但 `openaiBaseUrl` 指向智谱 → 再次失败 → `buildFallbackAnalysis()` → "AI服务暂不可用"。SiliconFlow 完全被跳过。

**根因：** `analyzeWithZhipu` 4 处 fallback 目标是 `analyzeWithOpenAI`，但 OpenAI 用的也是智谱后端，等于同后端重试。

**新 fallback 链（无循环）：**
```
智谱 → SiliconFlow → buildFallbackAnalysis
OpenAI → SiliconFlow → buildFallbackAnalysis  
MiMo → 智谱 → SiliconFlow → buildFallbackAnalysis
```

**修改项：**
1. `analyzeWithZhipu` — 4 处 `analyzeWithOpenAI` → `analyzeWithSiliconFlow`
2. `analyzeWithOpenAI` — 4 处 `buildFallbackAnalysis()` → `analyzeWithSiliconFlow(imageUrl)`
3. `analyzeWithSiliconFlow` — API Key 未配置时从 `analyzeWithOpenAI` 改为 `buildFallbackAnalysis`（防止循环）
4. 重启验证：`Started TuringDrawingApplication in 3.9s (port 8080)` ✅

## 编辑时分析被生图 provider 污染修复（22:44）

**问题：** 前端 `edit.vue` `selectedProvider='zhipu'`（生图用智谱），传到后端 `editImage()` 后，`analyzeImage(originalImage, useProvider)` 把 `'zhipu'` 作为分析 provider，导致直接走 `analyzeWithZhipu` 跳过了 MiMo。

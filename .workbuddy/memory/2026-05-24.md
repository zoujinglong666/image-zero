## /api/prompt/{id}/interact 500 错误修复（00:15）

### 问题链
1. `prompt_interactions` 表缺少 `updated_at` 字段（BaseEntity 要求）
2. `prompt_favorites / prompt_categories / vip_subscriptions / challenge_submissions / community_reports` 也缺少 `updated_at`
3. `SecurityConfig` 未放行 `/api/prompt/*/interact` 等路径，未登录用户返回 401
4. `PromptService.interact()` 未登录时传 `userId=0L` 写入 `prompt_interactions`，但 `user_id=0` 不满足 `users(id)` FK 约束

### 修复
- **V16** `db/migration/V16__add_missing_updated_at.sql`：给 5 张业务表补 `updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`
- **PromptService.java**：`interact()` 方法改为 `isGuest` 判断，未登录只增加计数，不写 `prompt_interactions` 记录
- **SecurityConfig.java**：`/api/prompt/*/interact`、`/api/prompt/*/favorite`、`/api/prompt/favorites/**`、`/api/prompt/mine/**` 加入 `permitAll()`

## 邀请码"加载中" + 创建订单 400 修复（10:52）

### 问题1：邀请码一直显示"加载中..."
**根因**：`http.interceptor.ts` 的 `response` 函数声明为 `async`，但 uview-pro 的 `Request.request()` 内部调用拦截器时没有用 `await`：
```typescript
const resInterceptors = this.interceptor.response(response); // 得到的是 Promise
if (resInterceptors !== false) {
    resolve(resInterceptors); // resolve 了一个 Promise 对象！
}
```
导致前端 `await getInviteInfo()` 得到的是 Promise 而非数据，`inviteCode` 为 undefined，始终显示"加载中"。

**修复**：`http.interceptor.ts` 第 110 行 `response: async (response: any)` → `response: (response: any)`（去掉 async）

### 问题2：创建订单返回 400 "创建订单失败，请稍后重试"
**根因**：`vip_subscriptions` 表 `expire_at DATETIME NOT NULL` 没有 DEFAULT 值，`PaymentService.createOrder()` 创建 pending 订单时未设置 `expire_at`，INSERT 直接报错。

**修复**：
- **V21** `db/migration/V21__fix_vip_subscription_expire_at.sql`：给 `expire_at` 添加 `DEFAULT CURRENT_TIMESTAMP`
- **PaymentService.java**：`createOrder()` 中 `VipSubscription.builder()` 增加 `.expireAt(LocalDateTime.now().plusDays(vipPlan.durationDays))`
